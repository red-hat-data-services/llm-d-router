Flow control now returns 503 (Service Unavailable) instead of 429 (Too Many Requests) when a request is rejected at queue capacity while the inference pool has no endpoints (e.g. scaled to zero), no longer signalling false partial availability. Requests dropped this way carry the header `x-llm-d-request-dropped-reason: rejected-no-endpoints`.
